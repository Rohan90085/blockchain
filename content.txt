delegatecall is used to execute another contract’s code but keep the current contract’s storage, address, and balance.
👉 Simple idea:
Copy code

Contract A uses code of Contract B
but
Storage remains in Contract A
2️⃣ Syntax
Solidity
Copy code

    contractAddress.delegatecall(data);
Usually written like:
Solidity
Copy code
(bool success, ) =
    contractAddress.delegatecall(
        abi.encodeWithSignature("set(uint256)",10)
    );
3️⃣ Example
Solidity
Copy code
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract A {
    uint public num;

    function set(uint _num) public {
        num = _num;
    }
}

contract B {
    uint public num;

    function setNum(address _contract, uint _num) public {
        (bool success, ) = _contract.delegatecall(
            abi.encodeWithSignature("set(uint256)", _num)
        );
        require(success);
    }
}

7️⃣ Important Interview Question
❓ Why is delegatecall used in proxy contracts?
Answer:
It allows executing logic from another contract while keeping storage in the proxy contract, enabling smart contract upgrades.
