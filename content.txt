Used to check condition
• Refunds remaining gas and reverts the transaction
Syntax:
Solidity
Copy code
require(condition, "message");
Used to
Validate user input
Check permissions
Prevent wrong transactions
Save gas fees
Example
Solidity
Copy code
function check(uint k) public view returns (string memory) {
    require(k > 100, "wrong");
    return "true";
}
Note:
If you give 99, an error will come because the condition k > 100 fails.
The transaction will revert and the default value will not be returned.

Contract Owner
Solidity
Copy code
contract Owner {
    address public owner;

    constructor() {
        owner = msg.sender;
    }

    function withdraw() public {
        require(msg.sender == owner, "not owner");
    }
}
Note:
If another person logs in, the operation will not be allowed.
Assert Error Handler
• Used to test conditions that should never be false
If it fails
Gas is consumed
Transaction fails
Indicates serious bug in contract
No custom message like require
Used for internal errors
No gas refund
Example
Solidity
Copy code
uint public count;

function increase() public {
    count = 1;
    assert(count > 0);
}
Note:
If count > 0 is true, the transaction continues.

Revert Handler
(Least used because we can use require)
Solidity
Copy code
function check(uint p) public {
    if (p < 10) {
        revert("less than p");
    }
}
Use:
Solidity
Copy code
revert("OK");
Modifiers
Modifiers are used to change the behavior of a function.
Uses
Access control (prevent unauthorized access)
Validation
Reusability of code
Improve contract security
How it works
Saves gas fees because if the condition fails, the transaction is cancelled.
